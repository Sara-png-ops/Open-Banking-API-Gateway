# API Rate Limiting

## Overview

Rate limiting protects APIs from abuse and ensures fair usage among developers.

---

## Limits

- 100 Requests per Minute
- 1000 Requests per Hour

---

## Policies

- IP-based limits
- Client-based limits
- API Key limits

---

## Response

When the limit is exceeded:

HTTP Status: 429 Too Many Requests

---

## Benefits

- Prevents abuse
- Improves stability
- Ensures fair access
